# Data Model Skill

## Data Pipeline
- `scripts/export_to_csv.py` parses `training_data/export.xml` -> per-user CSVs in `training_data/users/{uid}/`
- Extracts body metrics, daily aggregates (deduplicated Watch vs iPhone), swim events, km/mile splits
- **Source dedup**: `_deduplicate_records_by_source()` removes iPhone distance and StepCount records when Watch data exists
- **Apple splits**: `WorkoutEventTypeSegment` extracted into `.splits.json` (km + mile chains for run/bike/swim)
- **Incremental** via `.export_state.json`. Body metrics/daily aggregates always regenerated.
- Dashboard import: **copies** export.xml + workout-routes into `training_data/` (no symlinks)
- Per-user output: `IRONCOACH_OUT_DIR` env var (server sets to `training_data/users/{uid}/`)
- Run: `python3 scripts/export_to_csv.py` (add `--force` for full rebuild, defaults to user 1)

## CSV Files (training_data/)

73 columns. One row per workout.

Key columns:
| Column | Type | Notes |
|---|---|---|

| workoutActivityType | string | "HKWorkoutActivityTypeRunning" etc |

| startDate / endDate | datetime | "2026-03-10 09:08:42 +0200" |
| duration | float | **Seconds** |
| duration_min | float | Minutes (computed) |
| totalDistance | float | Raw distance (unit varies!) |
| distance_km | float | Normalized to km by server |
| HeartRate_average / _maximum | float | bpm |
| ActiveEnergyBurned_sum | float | Active calories (above BMR) |
| BasalEnergyBurned_sum | float | Resting calories |
| RunningSpeed_average | float | km/h |
| RunningPower_average | float | Watts |
| StepCount_sum | int | Total steps |
| CyclingSpeed_average | float | km/h |
| CyclingPower_average | float | Watts |
| CyclingCadence_average | float | rpm |
| DistanceSwimming_sum | float | **METERS** (not km!) |
| SwimmingStrokeCount_sum | int | Total strokes |
| meta_ElevationAscended | float | **CENTIMETERS** (├╖100 for meters) |

### Per-workout CSVs тАФ Time series
`workout_NNN_DATE_TYPE.csv` тАФ one row per data point (every ~1-5 seconds).

Common columns:
- `timestamp` тАФ datetime
- `Latitude`, `Longitude`, `Altitude` тАФ GPS
- `HeartRate` тАФ bpm at that instant
- `Speed` тАФ instantaneous speed
- `RunningPower`, `CyclingPower` тАФ watts
- `StepCount`, `SwimmingStrokeCount`
- `DistanceWalkingRunning`, `DistanceSwimming`, `DistanceCycling` тАФ cumulative

### Swim events JSON
`workout_NNN_DATE_Swimming.events.json`

Two event types:
1. **HKWorkoutEventTypeSegment** тАФ Apple Watch Auto Sets (rest-to-rest)
   - `date`: segment start time
   - `duration`: seconds of active swimming
   - `metadata.HKSWOLFScore`: real SWOLF per segment
   - `metadata.HKSwimmingStrokeStyle`: stroke code (0-6)
   - `metadata.HKLapLength`: pool length (typically "25 m")

2. **HKWorkoutEventTypeLap** тАФ Individual 25m laps
   - `date`: lap start time
   - `duration`: seconds for that lap
   - `metadata.HKSWOLFScore`: real SWOLF for that lap
   - `metadata.HKSwimmingStrokeStyle`: stroke code

## Per-User Files (training_data/users/{uid}/)

| File | Purpose | Generated By |
|:-----|:--------|:-------------|

### Per-Workout Files (workouts/YYYY-MM/)

| File | Purpose | Generated By |
|:-----|:--------|:-------------|
| `workout_NNN_*.csv` | Raw time-series (~3s resolution, 22 data columns) | export_to_csv.py |

| `workout_NNN_*.events.json` | Swim only тАФ lap/set events from Apple Watch | export_to_csv.py |

## Unit Reference (CRITICAL)

| Metric | Unit | Notes |
|---|---|---|
| Running distance | km | `DistanceWalkingRunning_sum` |
| Swimming distance | **meters** | `DistanceSwimming_sum` тАФ divide by 1000 for km |
| Cycling distance | km | `DistanceCycling_sum` |
| Elevation | **centimeters** | `meta_ElevationAscended` тАФ divide by 100 for meters |
| Duration (summary) | seconds | `duration` column |
| Duration (display) | minutes | `duration_min` column |
| HR | bpm | All HR fields |
| Power | watts | Running and cycling power |
| Calories | kcal | ActiveEnergyBurned = workout only, Basal = resting |
| Sleep | minutes | All sleep fields |
